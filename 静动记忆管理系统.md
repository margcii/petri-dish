# 静动记忆管理系统

## 系统概述

本系统管理 Cline AI 助手的记忆配置，采用**静态记忆**与**动态记忆**分离的架构，确保项目专属信息优先读取，通用经验作为补充。

---

## 记忆架构

### 一级：动态记忆（项目专属，最高优先级）

动态记忆存储在项目目录下的 `active_memory/` 文件夹，每个项目拥有独立的动态记忆空间。

**文件结构：**
```
d:\work\Engineering\projects\petri dish\
├── active_memory\
│   ├── dev_log/          # 项目架构和进度日志
│   │   └── [*.md]       # 项目各类文档
│   ├── rules/            # 项目专属约束准则
│   │   └── [*.md]       # 项目特定规则
│   └── workflows/        # 项目专属工作流程
│       └── [*.md]       # 项目特定流程
└── [项目文件]
```

**自动读取规则：**
- Cline 会自动读取所有 `active_memory/` 下的 `.md` 文件
- 项目专属配置已在此处注册，无需手动指定路径

**更新方式：**
- 项目结束后，可将通用内容迁移到静态记忆
- 项目进行中，随时添加/修改项目专属规则

---

### 二级：静态记忆（通用经验，次优先级）

静态记忆存储在 `D:\software\programming\static context\memory\`，作为跨项目的通用知识库。

**文件结构：**
```
D:\software\programming\static context\memory\
├── experience/           # 通用踩坑经验（自动读取）
│   └── [*.md]           # 通用问题解决方案
├── rules/                # 可复用约束准则（人工接入）
│   └── [*.md]           # 跨项目通用规则
└── workflows/            # 可复用工作流程（人工接入）
    └── [*.md]           # 跨项目通用流程
```

**自动读取规则：**
- ✅ **自动读取**：`experience/*.md` - 通用踩坑经验
- ❌ **不自动读取**：`rules/` 和 `workflows/` - 需人工明确指示才访问

**使用方式：**
- 通用经验自动生效
- 规则/流程复用需人工指定具体文件路径

**迁移流程：**
```
项目进行中 → 动态记忆.active_memory/rules/
    ↓
项目结束 → 评估通用性
    ↓
通用规则 → 静态记忆.static context/memory/rules/
```

## 使用流程

### 新项目初始化
1. 在项目根目录创建 `active_memory/` 目录结构
2. 初始化 `dev_log/`，记录项目架构说明
3. 将项目专属规则写入 `active_memory/rules/`

### 项目进行中
1. 根据需要读取 `active_memory/dev_log/` 了解项目进度
2. 根据需要读取 `active_memory/rules/` 了解项目约束
3. 如需通用经验，读取 `static context/memory/experience/*.md`

### 项目结束归档
1. 评估项目专用规则的通用性
2. 将通用规则迁移到 `static context/memory/rules/`
3. 更新全局配置中的工具信息
